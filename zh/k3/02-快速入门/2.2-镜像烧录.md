# 镜像烧录

## 1. 所需资源

| 类型 | 说明 |
| --- | --- |
| 硬件 | k3 开发板、Type-C 数据线、按键 |
| 刷机工具 | TiTan (2.1节提供)|
| 镜像 | [点击获取](https://spacemit.com/community/resources-download)|


**默认的开发板带镜像，请按需烧录**，推荐镜像 [LXQT rc4](https://archive.spacemit.com/image/k3/version/bianbu/v4.0rc1/Bianbu-LXQt-K3-v4.0rc1-20260408163758.tar.gz)


## 2. 操作步骤

按键参考
![按键](assets/com260_button.jpg)

### 2.1 烧录模式



- 准备一根 Type-C 数据线，一端连接至上位机 PC，另一端连接至 k3 开发板的 Type-C 接口；
- 按键分别接入 FC_REC - GND、SYS_RST - GND，如图
![按键接线](assets/button.jpg) 
- 长按 FC_REC 按键不松手；
- 短按一次 RST 按键；
- 松开 FC_REC 按键。

先按住 FC_REC 按键，再按下 RST 按键，即可进入刷机模式 


判断是否进入烧录模式
烧录工具：能扫描到设备
![](assets/found_device.jpg)

### 2.1 刷机



进入刷机模式后，参考 [刷机工具下载和使用指南](https://www.spacemit.com/community/document/info?lang=zh&nodepath=tools/user_guide/flasher_user_guide.md)，
选择推荐的资源镜像，点击开始刷机

## 3. 验证与自检

完成步骤后，用下列方式确认成功：

- 刷机软件
![success](assets/titan.jpg)

- 显示器

    COM260 预装 bianbu 操作系统，刷机后首次启动将运行配置向导, 通过鼠标与键盘注册用户名与密码

- 注
    如果无法外接显示器键盘进行首次注册，通过串口无法进入使用


    登录界面弹出

    ```
   Bianbu 4.0rc1 k3 ttyS0                                                             
    k3 login: 
                                                 
    密码：     
    ```
可以暂时使用超级用户开发

    用户： root

    密码： bianbu

建议不要用root用户登录。


## 4. 常见问题

刷机软件显示刷机失败：

1. 检查串口线是否连接正常，刷机过程中不要断开 USB 连接
2. 检查固件版本是否匹配
